## System Requirements

- **OS**: Windows 10 (build 17763) or later
- **Architecture**: x64 (64-bit)
- **Balatro Version**: Steam version (Xbox Game Pass is NOT supported)
- **WebView**: Automatically installed during setup

<Warning>
  The Balatro Mod Manager is **NOT compatible** with the Xbox Game Pass version of Balatro. You must own the Steam version.
</Warning>

## Installation

<Steps>
  <Step title="Download the Installer">
    Visit the [latest release page](https://github.com/skyline69/balatro-mod-manager/releases/latest) and download one of the Windows installers:

    - **EXE Installer** (recommended): `Balatro.Mod.Manager_..._x64-setup.exe`
    - **MSI Installer**: `Balatro.Mod.Manager_..._x64_en-US.msi`

    Both installers provide the same application; choose based on your preference.
  </Step>

  <Step title="Run the Installer">
    Double-click the downloaded installer file. Windows SmartScreen may display a warning.

    <Info>
      If SmartScreen blocks the installer:
      1. Click "More info"
      2. Click "Run anyway"

      All releases are [code-signed via SignPath](https://signpath.io) to verify authenticity.
    </Info>
  </Step>

  <Step title="Complete Installation">
    Follow the installation wizard:
    - Accept the license agreement
    - Choose installation location (default is recommended)
    - Select start menu folder
    - Click "Install"

    The installer will automatically download and install Microsoft Edge WebView2 if not already present.
  </Step>

  <Step title="Launch the Application">
    After installation completes, launch Balatro Mod Manager from:
    - Desktop shortcut (if created)
    - Start menu: `Balatro Mod Manager`
    - Install directory: `BMM.exe`
  </Step>
</Steps>

## File Locations

### Balatro Installation Detection

The mod manager automatically detects your Balatro installation by:

1. **Reading the Windows Registry** for Steam installation path:
   ```
   HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Valve\Steam\InstallPath
   ```

2. **Parsing Steam library folders** from:
   ```
   <Steam>\steamapps\libraryfolders.vdf
   ```

3. **Checking common locations**:
   ```
   C:\Program Files (x86)\Steam\steamapps\common\Balatro
   ```

<Note>
  If Balatro is not automatically detected, you can manually set the installation path in Settings.
</Note>

### Application Data Directories

| Data Type | Location |
|-----------|----------|
| **Configuration** | `%APPDATA%\Balatro\` |
| **Mods** | `%APPDATA%\Balatro\Mods\` |
| **Mod Manager Database** | `%APPDATA%\balatro-mod-manager\` |
| **Logs** | `%APPDATA%\balatro-mod-manager\logs\` |
| **Cache** | `%LOCALAPPDATA%\balatro-mod-manager\` |

<Tip>
  You can quickly access these folders by typing the path (e.g., `%APPDATA%\Balatro`) into Windows Explorer's address bar.
</Tip>

## Building from Source

### Prerequisites

Install the following tools before building:

<CodeGroup>
```powershell Chocolatey
choco install rust bun go-task
cargo install tauri-cli --version "^2.9"
```

```powershell Winget
winget install Rustlang.Rust.MSVC
winget install Oven-sh.Bun
winget install Task.Task
cargo install tauri-cli --version "^2.9"
```

```powershell Manual
# Install Rust
iwr https://win.rustup.rs -UseBasicParsing | iex

# Install Bun
powershell -c "irm bun.sh/install.ps1 | iex"

# Install Task
choco install go-task
# OR download from https://taskfile.dev/installation/

# Install Tauri CLI
cargo install tauri-cli --version "^2.9"
```
</CodeGroup>

### Automated Build

Run the automated installation script:

```powershell
iwr https://raw.githubusercontent.com/skyline69/balatro-mod-manager/main/scripts/install.ps1 -useb | iex
```

The script will:
- Clone the repository
- Install dependencies
- Build the frontend and backend
- Create the installer in `src-tauri/target/release/bundle/`

### Manual Build

<Steps>
  <Step title="Clone and Setup">
    ```powershell
    git clone https://github.com/skyline69/balatro-mod-manager.git
    cd balatro-mod-manager
    bun install --allow-scripts
    ```
  </Step>

  <Step title="Build Release">
    ```powershell
    task release-windows
    ```

    Or use the Bun commands directly:
    ```powershell
    bun run build
    cargo tauri build --target x86_64-pc-windows-msvc --verbose
    ```
  </Step>

  <Step title="Locate Build Artifacts">
    Built installers will be in:
    ```
    src-tauri\target\x86_64-pc-windows-msvc\release\bundle\nsis\*.exe
    src-tauri\target\x86_64-pc-windows-msvc\release\bundle\msi\*.msi
    ```
  </Step>
</Steps>

## Troubleshooting

### Windows SmartScreen Warning

**Issue**: SmartScreen blocks execution with "Windows protected your PC"

**Solution**:
1. Click "More info"
2. Verify the publisher shows the SignPath certificate details
3. Click "Run anyway"

Alternatively, right-click the `.exe` file and select "Run as administrator" after checking "More info".

### Steam Not Detected

**Issue**: Balatro installation path not found automatically

**Solution**:
1. Ensure Steam is installed and you've launched Balatro at least once
2. Check if `libraryfolders.vdf` exists:
   ```powershell
   Get-Content "C:\Program Files (x86)\Steam\steamapps\libraryfolders.vdf"
   ```
3. Manually set the Balatro path in Settings → Installation Path
4. Typical manual path:
   ```
   C:\Program Files (x86)\Steam\steamapps\common\Balatro
   ```

### WebView2 Installation Fails

**Issue**: Installer fails to download or install WebView2

**Solution**:
Manually install WebView2 Runtime:
1. Download from [Microsoft Edge WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)
2. Run the installer
3. Restart the Balatro Mod Manager installation

### Antivirus False Positives

**Issue**: Antivirus software flags the installer or application

**Solution**:
- Verify the digital signature (right-click → Properties → Digital Signatures)
- Add an exception for `BMM.exe` in your antivirus software
- Download only from official GitHub releases
- Check the [source code](https://github.com/skyline69/balatro-mod-manager) if concerned

### Application Won't Launch

**Issue**: Double-clicking the executable does nothing or crashes immediately

**Solution**:
1. Check Event Viewer for crash logs:
   - Press `Win + X` → Event Viewer
   - Navigate to Windows Logs → Application
   - Look for errors from "Balatro Mod Manager"

2. Verify WebView2 is installed:
   ```powershell
   Get-AppxPackage -Name "Microsoft.WebView2Runtime"
   ```

3. Run from command prompt to see error output:
   ```powershell
   cd "C:\Program Files\Balatro Mod Manager"
   .\BMM.exe
   ```

4. Check logs in:
   ```
   %APPDATA%\balatro-mod-manager\logs\
   ```

### Database Locked Errors

**Issue**: "Database is locked" errors when installing mods

**Solution**:
1. Close all instances of Balatro Mod Manager
2. End any background processes:
   ```powershell
   taskkill /F /IM BMM.exe
   ```
3. Delete the lock file (if it exists):
   ```powershell
   Remove-Item "$env:APPDATA\balatro-mod-manager\*.db-shm"
   Remove-Item "$env:APPDATA\balatro-mod-manager\*.db-wal"
   ```
4. Restart the application

## Advanced Configuration

### Custom Installation Path

If Balatro is installed in a non-standard location:

1. Open Balatro Mod Manager
2. Go to Settings → Installation Settings
3. Click "Browse" and select your Balatro installation folder
4. The folder should contain `Balatro.exe`

### Running Multiple Steam Libraries

The mod manager supports multiple Steam library folders automatically by parsing `libraryfolders.vdf`. All detected installations will be available in the installation path picker.

### Portable Installation

Balatro Mod Manager is not designed as a portable application. It stores data in standard Windows application directories (`%APPDATA%`). To use on multiple machines:

1. Install normally on each machine
2. Export mod collections from one installation
3. Import them on other installations

## Performance Tips

- **Disable Discord Rich Presence** if you don't use it (Settings → Discord)
- **Clear thumbnail cache** periodically (Settings → Cache)
- **Limit concurrent downloads** in Settings → Downloads
- Close Balatro before installing large mod packs for faster installation
